# 👁 Око мага — визуальный инспектор объектов C++

Учебная лаборатория: консольный инструмент, показывающий объект изнутри —
паспорт типа, поля с offset'ами, карту padding'а, vptr/vtable и сырые байты.

Рефлексии в C++ нет (до C++26), поэтому проект — экскурсия по всем легальным
и полулегальным способам вытащить из компилятора максимум. Каждый этап —
автономная программа в один `main.cpp`; финал собирает их приёмы
в header-only библиотеку с одним вызовом `eye::inspect(obj)`.

## Карта похода

| Этап | Папка | Что осваиваем |
|------|-------|---------------|
| M0 | `M0_passport/` | typeid + деманглер ABI, sizeof/alignof, type_traits |
| M1 | `M1_xray/` | hex-дамп байтов, endianness, SSO у std::string |
| M2 | `M2_anatomy/` | подсчёт полей агрегата, structured bindings, карта padding |
| M3 | `M3_vptr_lair/` | vptr, vtable, ручной виртуальный вызов, RTTI изнутри |
| M4 | `M4_guild_registry/` | макрос-реестр, указатели на члены, доступ к private |
| M5 | `M5_oracle/` | сверка показаний с `-fdump-lang-class` и pahole |
| Финал | `final/` | `magic_eye.hpp` — всё вместе, единый интерфейс |

**Порядок прохождения:** M0 → M1 → M2 → M3 → M4 → M5 → final.
Каждый следующий этап опирается на предыдущие. В каждой папке — свой README
с теорией, разбором и заданиями «проверь себя».

## Сборка

Всё сразу (из корня проекта):

```bash
cmake -B build
cmake --build build
ctest --test-dir build --output-on-failure
./build/m0_passport        # и остальные: m1_xray ... ex01_basics ...
```

Или любой этап одной командой (написана в шапке каждого main.cpp):

```bash
cd M2_anatomy
g++ -std=c++20 -Wall -Wextra -Wpedantic main.cpp -o m2 && ./m2
```

M5 собирается своим скриптом: `cd M5_oracle && ./oracle.sh` — там важен
не бинарник, а дамп компилятора.

## Платформы

Кроссплатформенно: **Linux/macOS (GCC/Clang)** и **Windows (MSVC / Visual
Studio)**. Платформенные различия спрятаны за `#ifdef` (единый шим в каждом
файле):

- **имя типа** — деманглер Itanium ABI `__cxa_demangle` на GCC/Clang;
  `typeid().name()` (уже читаемо) на MSVC;
- **цвета** — `isatty` (POSIX) либо `_isatty` + включение ANSI-консоли через
  WinAPI (Windows 10);
- **сырой разбор vtable** (M3 и секция vtable в финале) — только Itanium ABI
  (GCC/Clang). На MSVC объектная модель другая, поэтому там показывается
  динамический тип через `typeid` и честная заметка, а сырые ячейки не читаются.

**Windows.** Проще всего — открыть папку в Visual Studio (режим CMake): нужные
флаги `/utf-8 /Zc:preprocessor` уже в `CMakeLists.txt`. Вручную:

```bat
cl /std:c++20 /EHsc /utf-8 /Zc:preprocessor /I final final\examples\01_basics.cpp
```

`/utf-8` — исходники в UTF-8 (кириллица); `/Zc:preprocessor` — конформный
препроцессор для `EYE_DESCRIBE` (реестр работает и без него благодаря
`EYE_EXPAND`, но с флагом — по стандарту). Собрать под Linux можно и в Docker
без локального тулчейна: `docker run --rm -v "$PWD":/src -w /src gcc:13 bash -c
"g++ -std=c++20 -Wall -Wextra -Wpedantic M0_passport/main.cpp -o /tmp/m0 && /tmp/m0"`.

## Палитра

Золотой — заголовки, голубой — типы, зелёный — значения, тускло-красный —
padding, фиолетовый — vptr и прочая магия, серый — служебное. Цвета
отключаются сами, если вывод не в терминал (`./m2 > log.txt` даст чистый
текст) — за это отвечает `isatty`, разобран в M0.

## Честные границы

- **M2 (авторазбор)** работает только с агрегатами: никаких конструкторов,
  private, virtual и базовых классов. Лимит 8 полей (поднимается дописыванием
  веток в `visit_fields`). Битовые поля — нельзя. C-массив и вложенная
  структура считаются одним полем. Имена полей недоступны в принципе —
  компилятор их стёр (за именами — в M4).
- **M3 (vtable)** — исследование конкретной реализации (Itanium C++ ABI,
  GCC/Clang). По букве стандарта это UB. Инструмент обучения, не прод. Всё
  показанное сверяется с первоисточником в M5. Под MSVC этап компилируется,
  но сырой дамп vtable не строит (другая объектная модель) — печатает
  портируемую часть (динамический тип через typeid) и честную заметку.
- **M4 (реестр)** — регистрация руками: добавил поле в класс — добавь
  в `EYE_DESCRIBE`, иначе Око его не увидит.

## Куда смотреть после

- **boost::pfr** — промышленная версия M2 (авторазбор агрегатов)
- **boost::describe** — промышленная версия M4 (реестр макросами)
- **magic_enum** — тот же класс трюков, но для enum
- **C++26 static reflection (P2996)** — то, что сделает весь этот цирк
  историей. Понимать цирк всё равно полезно: он объясняет, почему рефлексию
  ждали тридцать лет.
