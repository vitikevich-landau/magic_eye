# M1 — Рентген памяти

## Цель

Увидеть объект глазами машины: плоский массив байтов. Никаких типов и
полей — только hex, offset'ы и ASCII-колонка, как в hex-редакторе.

## Теория в трёх абзацах

**Легальное чтение байтов.** Стандарт явно разрешает читать представление
любого объекта через `unsigned char*` (это одно из немногих исключений
из strict aliasing). Читать — можно. Писать через ручную арифметику
указателей — путь к беде, ты уже наступал на грабли `sizeof` vs `offsetof`
в fileshare. Здесь мы только смотрим.

**Endianness.** `0x11223344` лежит в памяти x86-64 как `44 33 22 11` —
младший байт по младшему адресу (little-endian). Порядок в памяти и
порядок «на проводе» (big-endian, network byte order) — разные вещи;
именно поэтому в fileshare ты руками собираешь кадры через
`write_u32be`, а не memcpy структуры в сокет.

**SSO — Small String Optimization.** Внутри `std::string` (libstdc++)
32 байта: указатель, длина и буфер на 15 символов. Короткая строка живёт
прямо в объекте — буквы видны в ASCII-колонке. Длинная переезжает в кучу,
и в объекте остаются только адрес и длина. Это же объясняет твой конспект
про `std::move`: у длинной строки «переместить» — значит скопировать
указатель, буквы в куче не шевелятся.

## Что смотреть в коде

- `xray()` — единственный `reinterpret_cast` этапа, и тот легальный.
- Байты padding'а в `ArmyStack` — мусор. Запусти дважды, сравни.
- Два дампа `std::string`: найди в первом слово, во втором — длину `0x2d`.

## Сборка

```bash
g++ -std=c++20 -Wall -Wextra -Wpedantic main.cpp -o m1 && ./m1
```

## Проверь себя

1. В дампе короткой строки по offset 0x0000 лежит какой-то адрес. Куда он
   указывает? (Подсказка: сравни с адресом самого объекта + 0x10.)
2. `double mana = 2.0;` — какие байты изменятся по сравнению с 1.0?
   Предскажи, потом проверь. (IEEE 754: меняется только экспонента.)
3. Почему нельзя сравнивать два `ArmyStack` через `memcmp`, даже если все
   поля равны?

## Связь с собесом

«Как object лежит в памяти», «что такое endianness и где стреляет»,
«почему memcmp для структур — баг» — плюс SSO, любимый вопрос про
внутренности std::string.
