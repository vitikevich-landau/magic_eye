# M0 — Паспорт типа

## Цель

Научиться спрашивать у компилятора всё, что он готов рассказать о типе
без единого хака: имя, размер, выравнивание и «анкету» из type_traits.

## Теория в трёх абзацах

**typeid и манглинг.** `typeid(T).name()` возвращает то же манглированное
имя, что ты видел в `nm`: `4Hero`, `St6vectorI...`. Внутри программы его
разворачивает `abi::__cxa_demangle` — это функция из Itanium ABI
(GCC/Clang), делает ровно то же, что `c++filt`. Она отдаёт строку из
`malloc`, поэтому в коде она завёрнута в `unique_ptr` с кастомным
deleter'ом `std::free` — маленький, но показательный приём RAII.

**type_traits.** Каждый трейт — вопрос компилятору, отвеченный на этапе
компиляции: `std::is_polymorphic_v<T>` («есть ли virtual → есть ли vptr»),
`std::is_aggregate_v<T>` («можно ли `T{a, b, c}`» — на этом стоит весь M2),
`std::is_trivially_copyable_v<T>` («можно ли memcpy» — важно для M1).
В бинарнике от них остаётся голая константа.

**Пустой тип весит 1 байт.** Не 0: два разных объекта обязаны иметь разные
адреса. `Tent a, b;` → `&a != &b`, а для этого каждому нужен хотя бы байт.

## Что смотреть в коде

- `clr::enabled()` — «магический статик»: инициализация один раз,
  потокобезопасно с C++11. Ты это уже проходил — вот живое применение.
- `isatty(fileno(stdout))` — как программа узнаёт, что её вывод
  перенаправили в файл, и гасит цвета.
- `passport<T>()` не принимает объект. Вообще. Вся информация существует
  до запуска программы.

## Сборка

```bash
g++ -std=c++20 -Wall -Wextra -Wpedantic main.cpp -o m0 && ./m0
```

## Что должно получиться (фрагмент)

```
== Creature ==========================================
  size / align          40 / 8
  polymorphic           нет
  aggregate             да
== Hero ==============================================
  size / align          16 / 8
  polymorphic           да
```

## Проверь себя

1. У `Hero` одно поле `int level` (4 байта), но `sizeof(Hero) == 16`.
   Откуда 16? (Разбивка: vptr (8) + `level` (4) + выравнивание (4). То есть
   скрытый указатель vptr — 8 байт, само поле — 4, и ещё 4 добивает
   компилятор до кратности 8. Полный разбор — M2 и M3.)
2. Почему `std::vector<Creature>` показывает `trivially copyable: нет`?
   Что сломается, если скопировать vector через memcpy?
3. Запусти `./m0 > log.txt` и открой файл. Куда делись цвета?

## Связь с собесом

«Что такое RTTI и сколько он стоит», «почему sizeof пустого класса не
ноль», «что значит trivially copyable» — всё это вопросы первого раунда.
Теперь у тебя на каждый есть программа, которая показывает ответ.
