# 01 · Видение и UX: как выглядит и ощущается «Странствие Ока»

## 1. Проблема текущего интерфейса

Сегодня `eye::inspect(obj)` печатает **всё сразу**: паспорт, иерархию, карту
памяти, vtable, спутники кучи. На большом объекте это простыня в несколько
экранов, по которой можно только прокручиваться терминалом. Указатели
показываются как адреса («Око покажет адрес, но значение читать не станет»),
и чтобы посмотреть, что по адресу, нужно вручную писать второй `inspect`.

Хочется другого опыта: **запустил программу — попал в обозреватель**.
Данные лежат деревом, по ним ходишь стрелками, раскрываешь то, что
интересно, по указателю **переходишь** и возвращаешься назад. Экран всегда
один, целиком твой, ничего никуда не уезжает.

## 2. Целевой опыт (elevator pitch)

> Как файловый менеджер, только вместо файлов — живые объекты C++.
> Midnight Commander по памяти процесса: слева дерево, справа —
> «Гримуар» выбранного узла, внизу — подсказки клавиш.

Принципы UX:

1. **Ноль обучения.** Всё управление — стрелки + Enter + Esc. Остальные
   клавиши — ускорители, и все они написаны в статус-баре. `?` — оверлей
   помощи.
2. **Ленивость.** Ничего не разыменовывается и не разбирается, пока
   пользователь не попросил. Раскрыл узел — тогда посчитали.
3. **Честность (как и раньше).** Непрозрачное — помечено непрозрачным,
   переход по указателю возможен только там, где Око знает тип и адрес
   ненулевой. Всё остальное — адрес и объяснение, почему дальше нельзя.
4. **Обратимость.** Любой переход отменяется: `←` сворачивает, `Backspace`
   возвращает по истории, `Esc` — выход без следов в терминале.
5. **Полный экран.** TUI открывается в альтернативном буфере на весь
   терминал; на Windows консоль по-прежнему авто-разворачивается
   (существующий `maximize_console_once`). После выхода терминал
   восстанавливается в точности (история прокрутки не теряется).

## 3. Публичный API

```cpp
#include <eye/magic_eye.hpp>

// Интерактивное странствие по одному объекту:
eye::explore(obj);
eye::explore(obj, "казна");        // своя подпись корня

// Несколько корней в одной сессии (галерея):
eye::Gallery g;
g.add(knight, "рыцарь");
g.add(nums,   "vector<int>");
g.add<Config>();                   // только статика типа (как inspect<T>())
g.run();                           // блокирует до выхода пользователя

// Старый API живёт без изменений:
eye::inspect(obj);                 // статическая печать, как сегодня
```

Правила деградации (подробно в 04): если stdout/stdin — не TTY, либо
`EYE_INTERACTIVE=0`, `explore` честно падает в `inspect` (печать всех
корней подряд). Так программы с `explore` остаются пригодными для CI и
redirect в файл.

## 4. Раскладка экрана

### 4.1 Широкий режим (две зоны, ширина ≥ 100 колонок)

```
┌ 👁 Око мага · странствие ── рыцарь ▸ banner ▸ буфер кучи ──────────────────┐
│                                          ║                                 │
│  ДЕРЕВО (фокус)                          ║  ДЕТАЛИ: banner · std::string   │
│                                          ║                                 │
│  ▾ 🏰 рыцарь : Knight            @0x7ffe │  паспорт ······················ │
│    ▾ ⛓ база Unit                 +0x00   ║   sizeof 32 · alignof 8         │
│        hp     : int = 100        +0x08   ║   не тривиально-копируем        │
│        speed  : int = 5          +0x0c   ║                                 │
│      armor    : int = 30         +0x10   ║  память ······················· │
│    ▾ banner   : std::string      +0x18   ║  0x18 ▓▓▓▓▓▓▓▓ e0 4a .. | .J..  │
│      → буфер кучи (7 Б)          @0x55e  ║  0x20 ▓▓▓▓▓▓▓▓ 07 00 .. | ....  │
│  ▸ ⚙ vptr → vtable Knight        +0x00   ║   ptr → 0x55e…  len 7  cap 15   │
│                                          ║   куча (не SSO)                 │
│  ▸ 📦 vector<int> nums           @0x7ffe │                                 │
│                                          ║  [h] hex   [v] vtable  [p] пасп │
├──────────────────────────────────────────╨─────────────────────────────────┤
│ ↑↓ выбор · →/Enter раскрыть/перейти · ← свернуть · ⌫ назад · Tab фокус ·   │
│ g к указателю · / поиск · f развернуть всё · ? помощь · q выход            │
└─────────────────────────────────────────────────────────────────────────────┘
```

- **Шапка** — breadcrumbs текущего пути (`рыцарь ▸ banner ▸ буфер кучи`),
  кликабельных мышью элементов нет — это чистая клавиатура.
- **Левая зона** — дерево навигации со скроллом; курсорная строка
  подсвечена инверсией. Возле каждого узла: пиктограмма вида, имя, тип,
  краткое значение, offset/адрес (правый край).
- **Правая зона** — панель деталей выбранного узла: те же секции, что
  рисует сегодняшний `inspect` (паспорт / память / vtable / спутники), но
  только для выбранного узла и с собственным скроллом.
- **Спайн `║`** — граница зон, уже есть в «Гримуаре v3»; Tab переносит
  фокус (фокусная зона помечена в заголовке зоны).
- **Гид** — 1–2 строки подсказок; сокращается на узких окнах.

### 4.2 Узкий режим (одна зона, < 100 колонок)

Дерево и детали становятся **двумя экранами**: дерево — главный,
`Enter` на листе открывает детали на весь кадр, `Esc`/`←` возвращает к
дереву. Порог соответствует существующему `two_zone`-порогу geometry.hpp.

### 4.3 Оверлеи

- `?` — помощь: полная карта клавиш поверх экрана (закрытие — любой клавишей).
- `/` — строка поиска внизу: инкрементальный фильтр по имени/типу поля,
  `Enter` — прыжок к первому совпадению, `n/N` — следующее/предыдущее.
- При ошибке перехода (nullptr, цикл, непрозрачный тип) — однострочный
  тост в статус-баре, не модальное окно.

## 5. Карта клавиш

| Клавиша | Действие |
|---|---|
| `↑` / `↓` (`k` / `j`) | курсор по дереву вверх/вниз |
| `→` / `Enter` (`l`) | раскрыть узел; на указателе — **перейти** к pointee; на листе в узком режиме — открыть детали |
| `←` (`h`) | свернуть узел; если уже свёрнут — подняться к родителю |
| `Backspace` / `b` | назад по истории переходов |
| `Tab` | переключить фокус дерево ↔ детали (широкий режим) |
| `PgUp`/`PgDn`, `Home`/`End` | прокрутка фокусной зоны |
| `g` | перейти по указателю под курсором (синоним `Enter` на указателе) |
| `p` / `m` / `v` / `x` | режим правой панели: паспорт / память / vtable / чистый hex |
| `f` | развернуть/свернуть длинные регионы (живой аналог `EYE_FULL`) |
| `e` | раскрыть всех детей узла рекурсивно (до лимита глубины) |
| `c` | свернуть всё до корней |
| `/`, `n`, `N` | поиск, следующее/предыдущее совпадение |
| `1..9` | прыжок к N-му корню галереи |
| `?` / `F1` | помощь |
| `q` / `Esc` | выход (Esc сначала закрывает оверлей/поиск, если открыты) |

`hjkl` — дублёры для vim-рук; в помощи упомянуть, в гиде не показывать.

## 6. Дерево: что и как показывается

Виды узлов и их строки (пиктограмма — один символ из уже используемого
Unicode-набора, деградирует в ASCII при `EYE_ASCII=1`):

| Узел | Строка в дереве | Раскрытие даёт |
|---|---|---|
| корень-объект | `🏰 имя : Тип @адрес` | базы, поля, vptr, спутники |
| под-объект базы | `⛓ база Unit +0x00 [virtual] [общая]` | поля базы, её vptr |
| поле-скаляр | `hp : int = 100 +0x08` | лист (детали справа) |
| поле-указатель | `power : int* → 0x7ffe… +0x08` | **переход** к pointee (если тип известен и не nullptr) |
| nullptr-указатель | `nothing : void* → ∅` | лист; тост «переход невозможен: nullptr» |
| указатель внутрь себя | `self : Scroll* → база+0x00 ⟲` | переход = прыжок курсора на уже существующий узел |
| поле-строка | `banner : std::string "Griffin" (7/15)` | узел SSO-буфера или спутник кучи |
| std::vector | `📦 nums : vector<int> size 3 cap 4` | служебные слова + узел «элементы (куча)» |
| элементы вектора | `→ элементы[3] @0x55e…` | элементы `#0, #1, …` (страницами по 100) |
| vptr-сайт | `⚙ vptr → vtable Knight +0x00` | детали vtable справа (offset-to-top, слоты) |
| спутник (буфер строки) | `→ буфер кучи (34 Б) @0x55e…` | hex-превью буфера справа |
| padding | в дереве **не показывается**; виден на карте памяти справа | — |
| непрозрачное | `▨ скрыто: OpaqueBase (24 Б)` | лист + совет `EYE_DESCRIBE` справа |

Сортировка детей — по offset (как сейчас). Значения в строках дерева —
короткие превью (обрезка `clip`), полная версия — в панели деталей.

## 7. Пользовательские сценарии (приёмочные)

**С1 — охота на padding.** Пользователь запускает `ex_02`, видит агрегат,
стрелками спускается к полю, справа на карте памяти подсвечен регион этого
поля; между полями видит дыру и причину («требует адрес, кратный 8»).
Не покидая программы, смотрит все три объекта примера (клавиши 1–3).

**С2 — путешествие по указателю.** В `ex_10` курсор на `power : int*`,
`Enter` → детали `int` со значением 350 по адресу; breadcrumbs показывает
`свиток ▸ power ▸ *power`; `Backspace` возвращает к свитку. На `nothing`
(nullptr) `Enter` даёт тост, никуда не переходит. На `self` — прыжок на
корневой узел свитка с пометкой цикла `⟲`.

**С3 — строка и куча.** В `ex_10` `Grimoire::spell` раскрывается в узел
«буфер кучи», при выборе справа — hex-дамп кучного буфера (сегодняшний
спутник), у `owner` (SSO) — буфер внутри объекта, спутника нет и это
объяснено.

**С4 — наследование.** В `ex_08` (ромб) дерево показывает вложенность баз
с пометками `[virtual]`/`[общая]`, у каждого полиморфного под-объекта — свой
vptr-узел; выбор vptr показывает справа блок-диаграмму vtable с
offset-to-top.

**С5 — некуда смотреть.** `./ex_10 > log.txt` — программа не зависает в
ожидании клавиш, а печатает статический `inspect` всех корней (текущий
формат). CI остаётся зелёным без псевдотерминала.

**С6 — большой vector.** `std::vector<int>` на 10 000 элементов: узел
«элементы» раскрывается страницами («#0…#99», узел «⋯ ещё 9900 — Enter»),
интерфейс не замирает.
