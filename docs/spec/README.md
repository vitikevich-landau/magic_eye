## Резюме в трёх абзацах

**Что строим.** Новый публичный вход `eye::explore(obj)` (и `eye::Gallery`
для нескольких корней): полноэкранное TUI-приложение в альтернативном буфере
терминала. Слева — дерево навигации (объект → базы → поля → указатели →
спутники кучи), справа — панель деталей выбранного узла (паспорт, карта
памяти в стиле «Гримуар v3», hex, vtable). Управление — стрелки/Enter/Esc,
переход по указателю — как переход по ссылке в браузере, с историей «назад».

**Что для этого меняем.** Вид перестаёт печатать в `std::cout` напрямую и
начинает рисовать в буфер строк (`Surface`) — статический `eye::inspect()`
просто сливает буфер в поток (вывод байт-в-байт прежний, это проверяют
регрессии), а TUI кладёт тот же буфер в прокручиваемую панель. Модель
получает надстройку — **навигационный граф** `NavNode` с ленивыми детьми:
раскрытие узла и разыменование указателя происходят только по запросу
пользователя, с защитой от `nullptr`, циклов и глубины.

**Чего не меняем.** `eye::inspect()` остаётся и работает как раньше
(отчёты, дифф-логи, CI). Библиотека остаётся header-only, C++20, без
внешних зависимостей. Граница «модель ↔ вид» остаётся проверяемой
компилятором; появляется третий слой «навигация» с явно описанными правами.

## Быстрый словарь

- **Узел (NavNode)** — вершина навигационного графа: объект, под-объект базы,
  поле, vptr-сайт, спутник кучи, элементы vector, pointee указателя.
- **Surface** — буфер стилизованных строк, в который рисуют `view_*`;
  единица обмена между видом и экраном.
- **Канва (Canvas)** — экранный буфер ячеек TUI с дифф-отрисовкой.
- **Гид** — статус-бар с подсказками клавиш внизу экрана.
- **Странствие** — сессия навигации: путь от корня + история переходов.
