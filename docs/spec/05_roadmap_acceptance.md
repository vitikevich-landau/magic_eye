# 05 · Дорожная карта, приёмка, тестирование, риски

## 1. Этапы (каждый — отдельный PR, зелёный сам по себе)

### M-A · Шов Surface (рефакторинг вида, поведение не меняется)

- `detail/surface.hpp`: StyledLine, Surface, active_surface, SurfaceScope.
- `frame.hpp::put` и семейство `frame_*` пишут в активный Surface либо в
  cout (как раньше).
- `GeoScope` в geometry.hpp (временная ширина рамки).
- **Приёмка:** все существующие тесты и все `ex_01…ex_10` дают вывод
  байт-в-байт как до правки (снять эталоны до начала работ:
  `EYE_WIDTH=126 EYE_COLOR=1 ./ex_N` и `EYE_COLOR=0`-варианты).

### M-B · Ядро TUI (терминал, клавиши, канва) — без навигации

- `tui/term.hpp`, `tui/keys.hpp`, `tui/canvas.hpp`.
- Демо-приложение в tests/ (не в examples/): рисует рамку с «hello»,
  реагирует на стрелки, resize, q. Ручная проверка по матрице терминалов.
- **Приёмка:** юнит-тесты декодера клавиш (байты→события, таблица из
  04 §3, включая расчленённые по read'ам последовательности); терминал
  восстанавливается после q / Ctrl-C / kill -TERM (ручной чек-лист);
  0% CPU в простое (`top`).

### M-C · Навигация: дерево + детали (без переходов по указателям)

- `nav/nav_node.hpp`, `nav/nav_build.hpp`, `nav/nav_session.hpp`,
  `tui/widgets.hpp`, `tui/app.hpp`, `explore.hpp`, `eye::explore/Gallery`.
- Дерево: корни, базы, поля, vptr-сайты, спутники, элементы vector с
  пагинацией. Детали: режимы память/паспорт/vtable/hex через SurfaceScope.
- Деградация не-TTY → печать (сценарий С5).
- **Приёмка:** сценарии С1, С3, С4, С6 из 01 §7 на `ex_02/08/10`;
  снапшот-тесты кадров (см. §2); include-тест «magic_eye.hpp без вызова
  explore не инстанцирует tui/nav» (компиляция файла с одним inspect —
  символы tui в объектнике отсутствуют).

### M-D · Переходы: указатели, история, циклы

- Thunk-реестр в reflect_impl (03 §3), pointee-узлы, history/Backspace,
  интервальная карта «указатель внутрь себя ⟲», умные указатели,
  breadcrumbs в шапке.
- **Приёмка:** сценарий С2 целиком; тест цикла (связный список из 3 узлов:
  переходы не плодят дубликатов); nullptr/`void*`/`char*` дают
  `follow_block_reason`, не переход; УБ-санитары чисты на ex_10-странствии
  под EYE_SCRIPT.

### M-E · Полировка

- Поиск `/` n N, помощь `?`, `e`/`c` (рекурсивное раскрытие/свёртка),
  `1..9` — корни галереи, тосты, EYE_ASCII, живое обновление значений
  (03 §5), Windows-прогон (обе консоли), README и новый пример
  `examples/11_explorer.cpp` (галерея из объектов примеров 05+10).
- **Приёмка:** полный чек-лист §3; README описывает explore;
  `ex_11` — витрина.

Зависимости: M-A → M-C; M-B → M-C; M-C → M-D → M-E. M-A и M-B независимы,
можно параллельно.

## 2. Тестирование

**Юнит (ctest, без терминала):**
- декодер клавиш: таблица байтов → события; ESC-одиночка против ESC-хвоста;
  UTF-8 по частям; мусорные CSI;
- NavSession: expand/collapse/visible-проекция, курсор у краёв, пагинация
  more, история, ключи циклов;
- канва: blit с клипом ANSI-строк (цвет закрыт на границе), дифф отдаёт
  только изменённые строки.

**Снапшот-кадры (главный приёмочный инструмент):**
режим `EYE_SCRIPT`: `EYE_INTERACTIVE=script EYE_WIDTH=126 EYE_HEIGHT=40
EYE_SCRIPT="down down right q" ./ex_10` — приложение не трогает терминал,
исполняет клавиши из строки и печатает каждый кадр канвы в stdout с
разделителем `── frame N ──`. Эталоны лежат в tests/, сравнение — как в
существующих регрессиях. Это же — путь отладки «что нарисовалось».

**Регрессия печати:** статические эталоны inspect (снятые перед M-A)
проверяются на каждом этапе.

**Ручной чек-лист (перед M-E-приёмкой):** матрица терминалов —
gnome-terminal/kitty/xterm/tmux (Linux), Terminal/iTerm2 (macOS, по
возможности), Windows Terminal + классический conhost (Windows): вход/выход
чистые, стрелки/PgUp/resize/Ctrl-C, кириллица в поиске, EYE_COLOR=0.

**Санитары:** ASan/UBSan-прогон EYE_SCRIPT-сценариев в CI (как сейчас
для тестов).

## 3. Сводные критерии приёмки фичи

1. `eye::explore(obj)` открывает полноэкранный TUI; выход возвращает
   терминал в исходное состояние во всех случаях завершения.
2. Все шесть сценариев 01 §7 проходят.
3. `eye::inspect` даёт вывод байт-в-байт как до фичи.
4. Не-TTY/CI: программы с explore завершаются сами, печатая статический
   осмотр.
5. Header-only, C++20, GCC 13 (-Wall -Wextra -Wpedantic, ASan/UBSan чисто)
   и MSVC (/W4 /utf-8 /Zc:preprocessor) — собирается и работает.
6. Программа только с inspect не платит за TUI (ни инстанцирования
   tui-шаблонов, ни поведенческих изменений).

## 4. Риски и решения

| Риск | Вероятн. | Решение |
|---|---|---|
| Клип ANSI-строк на границе зон (протекание цвета, разрезанные широкие символы) | высокая | вынести клип в text.hpp, покрыть юнитами до M-C; всегда закрывать `\x1b[0m` |
| VT-input отсутствует в старом conhost | средняя | фолбэк ReadConsoleInput (04 §2); матрица ручных прогонов |
| view_memory не готов к произвольной ширине зоны | средняя | GeoScope + прогон всех примеров через ScrollPane на M-C; в крайнем случае — минимальная ширина зоны деталей = MIN_FRAME_W с гориз. усечением |
| std::function-thunks утяжеляют компиляцию inspect-only программ | средняя | точка кастомизации 03 §3.2: без explore thunks не инстанцируются; include-тест в CI |
| Разыменование указателей — UB при висячих ссылках | принято | по клику пользователя, с документированным контрактом (объекты живы на время run) — тот же класс честности, что vtable-разбор |
| ConPTY/IDE-терминалы с размером 0×0 | низкая | минимальный размер 60×15, иначе деградация в печать |
| SIGTSTP/SIGCONT ломает raw-режим | низкая | обработчик SIGCONT: повторная инициализация (04 §7) |

## 5. Вне рамок (осознанно не делаем сейчас)

- Мышь (клики по дереву) — возможное продолжение, протокол SGR 1006.
- Редактирование памяти из TUI — никогда: Око смотрит, не трогает.
- Watch-режим (авто-перерисовка по таймеру при изменении объекта другим
  потоком) — после M-E, если понадобится.
- Дамп текущего экрана в файл (`s` — снимок) — кандидат в M-E+, дёшев
  благодаря канве.
- Поддержка терминалов без ANSI (голый cmd.exe до Win10) — деградация в
  печать, TUI не обещаем.
