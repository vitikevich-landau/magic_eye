# 04 · Терминальный слой: raw-режим, клавиши, канва, кроссплатформенность

## 1. Принципы

- Без внешних библиотек: POSIX termios + ANSI/VT на Linux/macOS,
  WinAPI + VT-вывод на Windows. `<windows.h>` — по-прежнему только через
  platform.hpp.
- **Терминал возвращается в исходное состояние всегда**: нормальный выход,
  `Ctrl-C`, исключение. RAII + обработчики сигналов.
- Никакой фоновой активности: цикл спит в ожидании ввода; CPU в покое ~0%.

## 2. detail/tui/term.hpp — сессия терминала

```cpp
class TermSession {                 // RAII на всё странствие
public:
    TermSession();                  // войти (см. последовательность ниже)
    ~TermSession();                 // выйти (восстановить ВСЁ)
    static bool interactive_capable();  // можно ли вообще TUI (см. §6)
    Size size() const;              // текущие колонки×строки
    bool poll_key(Key& out, int timeout_ms);  // клавиша или таймаут
};
```

Вход (POSIX):
1. `tcgetattr` — сохранить исходные termios.
2. raw-режим: `~(ICANON|ECHO|ISIG…)` (ISIG выключаем: Ctrl-C
   обрабатываем сами как выход — так гарантируем восстановление;
   Ctrl-Z/SIGTSTP — оставить и обработать SIGCONT повторной инициализацией).
3. Альтернативный экран `\x1b[?1049h`, спрятать курсор `\x1b[?25l`.
4. Установить обработчики `SIGTERM`/`SIGHUP` (восстановить терминал и
   умереть) и `SIGWINCH` (флаг resize; сам пересчёт — в цикле).
   Обработчики — с сохранением/восстановлением прежних.

Вход (Windows):
1. Сохранить `GetConsoleMode` для stdin и stdout.
2. stdin: снять `ENABLE_LINE_INPUT|ENABLE_ECHO_INPUT|ENABLE_PROCESSED_INPUT`,
   поставить `ENABLE_VIRTUAL_TERMINAL_INPUT` (стрелки приходят
   VT-последовательностями — декодер один на обе платформы; фолбэк —
   `ReadConsoleInput` с KEY_EVENT, если VT-input недоступен: conhost
   до Win10 1607 не умеет).
3. stdout: `ENABLE_VIRTUAL_TERMINAL_PROCESSING` (вывод уже ANSI);
   alt-screen тем же `\x1b[?1049h`.
4. `maximize_console_once()` — существующий разворот на весь экран
   (уважает `EYE_RESIZE=0`).

Выход (обе платформы, и в деструкторе, и в сигнальных обработчиках):
показать курсор, `\x1b[?1049l`, вернуть termios/ConsoleMode. Выходная
последовательность продублирована в `std::atexit`-страховке с
идемпотентным флагом.

## 3. detail/tui/keys.hpp — декодер ввода

```cpp
enum class Key {
    none, up, down, left, right, pgup, pgdn, home, end,
    enter, backspace, tab, esc, f1, chr /* + поле unicode-символа */,
    resize,                       // синтетическая: изменение окна
};
struct KeyEvent { Key key; char32_t ch; };
```

Декодер — конечный автомат над байтовым потоком:

- `\x1b` в одиночку против `\x1b[A`: после ESC ждём хвост ≤ 25 мс
  (poll с таймаутом); нет хвоста — это клавиша Esc.
- Поддерживаемые последовательности: CSI (`\x1b[A/B/C/D`, `\x1b[5~/6~`,
  `\x1b[H/F`, `\x1b[1~/4~`), SS3 (`\x1bOA…`, старые xterm), `\x7f`/`\x08`
  (backspace), `\r`/`\n` (enter), `\t`. Всё прочее CSI — молча съесть до
  финального байта (не мусорить в поиск).
- UTF-8: многобайтовые символы собираются в `char32_t` (для строки поиска
  и русских букв в ней).
- resize: POSIX — флаг из SIGWINCH проверяется на каждом обороте;
  Windows — WINDOW_BUFFER_SIZE_EVENT из ReadConsoleInput, а в VT-режиме —
  сравнение размера при каждом таймауте poll (100 мс).

Модуль обязан быть **чистой функцией над байтами** (кормим буфер —
получаем события): это единственный способ юнит-тестировать его без
терминала (см. 05, тесты).

## 4. detail/tui/canvas.hpp — канва и дифф-отрисовка

```cpp
struct Canvas {
    void resize(Size s);
    void clear();
    // положить готовую ANSI-строку (StyledLine) в позицию зоны с клипом:
    void blit(std::size_t row, std::size_t col, const StyledLine& l,
              std::size_t max_width);
    void frame();                  // вывести дифф с прошлым кадром
};
```

- Хранение: по строкам, строка = собранный `std::string` с ANSI +
  печатная ширина. Дифф — построчный (строка изменилась → перерисовать
  строку целиком через `\x1b[row;1H`), посимвольный дифф не нужен.
- Вывод кадра — одним `write` из собранного буфера (без мигания;
  `std::cout` с выключенной синхронизацией или прямой write в fd).
- Клип по печатной ширине с учётом ANSI-эскейпов и широких символов —
  переиспользуем/обобщаем `text.hpp` (там уже есть учёт ширины и clip).
  Это самая тонкая часть канвы: обрезка строки с эскейпами обязана
  закрывать цвет (`\x1b[0m` в конце клипа) — иначе «протечёт» в соседнюю
  зону.

## 5. detail/tui/widgets.hpp и app.hpp

Виджеты — минимальные, без фреймворка:

- **TreeView**: `NavSession.visible` + окно скролла → строки в канву;
  курсор — инверсия (`\x1b[7m`); авто-подскролл, чтобы курсор был виден;
  колонка offset/адрес прижата к правому краю зоны.
- **ScrollPane**: Surface (детали узла) + смещение; индикатор
  «строки M–N из K» в заголовке зоны.
- **StatusBar**: гид по клавишам (сокращается по ширине: полный →
  средний → «? помощь»); туда же — тосты с таймаутом 3 с (по таймауту
  poll).
- **HelpOverlay / SearchBar** — поверх канвы, забирают ввод на себя.

**app.hpp — цикл:**

```cpp
void run_gallery(std::vector<NavNode> roots) {
    if (!TermSession::interactive_capable()) { print_all_static(roots); return; }
    TermSession term;
    NavSession nav{std::move(roots)};
    Layout lay;  Canvas canvas;
    for (bool alive = true; alive;) {
        lay = compute_layout(term.size());       // зоны: дерево│детали│гид
        draw(canvas, lay, nav);                  // дерево, детали, гид, оверлеи
        canvas.frame();
        KeyEvent e;
        if (!term.poll_key(e, 100)) continue;    // таймаут: тосты/resize
        alive = dispatch(e, nav, lay);           // таблица клавиш из 01 §5
    }
}
```

Раскладка зон считается от `term.size()` теми же порогами, что
`geo_refresh` (two_zone): широкий режим — дерево 45% ║ детали 55%
(дерево не уже 38 колонок), узкий — один экран (01 §4.2). Высота: шапка 1,
гид 1–2, остальное — зоны.

## 6. Деградация и окружение

`TermSession::interactive_capable()` — TUI возможен, если:

- `isatty(stdin) && isatty(stdout)` (POSIX) / консольные хэндлы (Win);
- `TERM != "dumb"` (POSIX);
- `EYE_INTERACTIVE` не `0`.

Иначе `explore`/`Gallery::run` печатают статические `inspect` всех корней
(сценарий С5). Отдельно: `EYE_INTERACTIVE=0` — принудительная печать даже
в терминале (для скриптов).

Новые переменные окружения (дополняют существующие):

| переменная | что делает |
|---|---|
| `EYE_INTERACTIVE=0` | не входить в TUI, печатать как inspect |
| `EYE_ASCII=1` | пиктограммы дерева и рамки — чистый ASCII (бедные шрифты) |
| `EYE_SCRIPT="j j l q"` | скормить клавиши из строки и печатать кадры в stdout (тестовый режим, см. 05) |

Существующие переменные продолжают работать: `EYE_COLOR` (канва без
цвета), `EYE_FULL` (стартовое состояние `f`), `EYE_RESIZE`
(Windows-разворот), `EYE_WIDTH` (перекрывает размер и в TUI — полезно
для снапшот-тестов вместе с новой `EYE_HEIGHT`).

## 7. Известные терминальные грабли (проверить на M-B)

- tmux/screen: alt-screen и 1049 работают, но `TERM=screen-256color` —
  проверить палитру; SS3-стрелки.
- Windows Terminal против старого conhost: VT-input есть/нет — фолбэк на
  ReadConsoleInput обязателен; разворот окна WT игнорирует (уже
  задокументировано в geometry.hpp — ок).
- ConPTY (запуск из IDE): isatty истинен, но размеры могут прийти 0×0 —
  защита минимальным размером 60×15, ниже — честный отказ в TUI с печатью.
- Широкие символы (пиктограммы, кириллица) в клипе — на границе зоны не
  резать пополам (уже решается text.hpp, перепроверить на канве).
- SIGTSTP (Ctrl-Z) + fg: повторная инициализация raw-режима по SIGCONT.
